📖 Project Overview

Finals - Lab Quiz 1: Sentiment Analysis - POSITIVE

Brief Description: This project implements a Recurrent Neural Network (RNN) using LSTM layers to perform sentiment analysis on movie reviews from the IMDB dataset. The model classifies text as either POSITIVE or NEGATIVE sentiment. The trained model is deployed as an Anvil web app for real-time sentiment predictions.

Objectives / Purpose: Build an RNN model that can accurately classify sentiment from user input text, deploy it as a web application, and demonstrate correct POSITIVE sentiment prediction.

Context Note: This web app will also be used for Finals - Lab Quiz 2 (Sentiment Analysis - NEGATIVE) to demonstrate NEGATIVE sentiment prediction.

✨ Key Features

  • RNN with LSTM: Long Short-Term Memory layer captures sequential dependencies in text.
  • Embedding Layer: Converts words to dense vector representations (128 dimensions).
  • IMDB Dataset: 20,000 most frequent words, sequences padded to 100 length.
  • Binary Classification: Sigmoid activation output (Positive > 0.5, Negative < 0.5).
  • Anvil Deployment: Web app with text input and real-time sentiment prediction.

💻 Important Code Blocks

1. RNN Model Architecture (LSTM)

Embedding layer → LSTM layer → Dense output layer for binary sentiment classification.


model = tf.keras.Sequential()

# Embedding Layer: converts words to 128-dim vectors
model.add(tf.keras.layers.Embedding(vocab_size, embed_size, input_shape=(X_train.shape[1],)))

# LSTM Layer: 128 units with tanh activation
model.add(tf.keras.layers.LSTM(units=128, activation='tanh'))

# Output Layer: sigmoid for binary classification
model.add(tf.keras.layers.Dense(units=1, activation='sigmoid'))

model.compile(optimizer='rmsprop', loss='binary_crossentropy', metrics=['accuracy'])

🧩 Challenges & Solutions

Challenge 1: Text inputs vary in length; neural networks require fixed input sizes.

Solution: Used pad_sequences() to pad or truncate all sequences to max_len=100.

Challenge 2: Raw text needs conversion to numerical sequences for model input.

Solution: Used IMDB's word_index mapping to convert words to integers (+3 offset for special tokens).

Challenge 3: Deploying to Anvil required handling text input from web interface.

Solution: Created @anvil.server.callable function that preprocesses text and returns sentiment label.

🧠 Learning Reflection

What I learned: This project taught me how RNNs with LSTM layers capture sequential dependencies in text data. Unlike CNNs or dense networks, LSTMs remember information over longer sequences, making them ideal for sentiment analysis where word order matters. I also learned about tokenization, padding, and embedding layers for text preprocessing.
