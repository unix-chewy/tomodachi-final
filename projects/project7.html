📖 Project Overview

Boston House Price Prediction

Brief Description: This project implements a Neural Network regression model to predict house prices using the Boston Housing dataset. As a Junior Data Scientist for a real estate development company, the goal is to identify key features influencing house prices and build a deployable prediction model. The model was trained in Google Colab and is prepared for full-stack deployment via Anvil.works.

Objectives / Purpose: Identify 3 most relevant features using correlation heatmap, build a simple NN with Keras/TensorFlow, train/evaluate the model, and deploy as a web app where users input feature values to get real-time price predictions.

Context Note: This project serves as the foundation for Midterm - Hands-on Quiz 1 (House Price Prediction), where the model will be deployed as a fully functional web application using Anvil.works.

✨ Key Features

  • Feature Selection via Correlation Heatmap: Used seaborn heatmap to identify top 3 features correlated with house price (Istat, ptraito, indus).
  • Neural Network Architecture: 3 layers — Dense(16, ReLU), Dense(8, ReLU), Dense(1) output layer.
  • Feature Scaling: StandardScaler applied to normalize input features before training.
  • Training Analysis: Compared loss values at 100 epochs (Train: 15.43, Val: 20.59) vs 300 epochs (Train: 13.31, Val: 18.21).
  • Anvil Deployment Ready: Model designed to be connected to Anvil.works web app with form inputs for 3 features. Deployment to be completed in Midterm - Hands-on Quiz 1.

2. Neural Network Model Definition

Sequential model with 2 hidden layers (16 and 8 neurons, ReLU activation) and output layer for regression.


model = Sequential()
model.add(Dense(16, activation='relu', input_shape=(X_train.shape[1],)))
model.add(Dense(8, activation='relu'))
model.add(Dense(1))
model.compile(optimizer='adam', loss='mse')

3. Model Training with Validation Split

Trains model for 100 epochs (extended to 300 for comparison) with 20% validation split.


history = model.fit(

5. Anvil Deployment Preparation (For Midterm - Hands-on Quiz 1)

Code structure for connecting trained model to Anvil web app.


import anvil.server
anvil.server.connect("YOUR_ANVIL_SERVER_KEY")

@anvil.server.callable
def predict_price(feature1, feature2, feature3):
    scaled_input = scaler.transform([[feature1, feature2, feature3]])
    prediction = model.predict(scaled_input)
    return float(prediction[0][0])

anvil.server.wait_forever()

This will be connected to an Anvil form with three input fields displaying the predicted house price.

🧩 Challenges & Solutions

Challenge 1: Raw features not scaled — Neural Networks are sensitive to input magnitude differences.

Solution: Applied StandardScaler to normalize all features (mean=0, std=1) before training.

Challenge 2: Determining optimal number of epochs — validation loss started increasing after certain point (overfitting).

Solution: Compared 100 epochs (Train loss: 15.43, Val loss: 20.59) vs 300 epochs (Train: 13.31, Val: 18.21). While both decrease, plotting loss curves helps detect overfitting visually.

Upcoming Challenge (Hands-on Quiz 1): Deploying model to Anvil.works as a functional web app.

Planned Solution: Use anvil.server.callable decorator to expose prediction function, create form with three input fields, and display prediction output.
