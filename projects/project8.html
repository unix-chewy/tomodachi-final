📖 Project Overview

Improving Your Boston Housing Price Prediction Model

Brief Description: This project explores techniques to improve the Boston House Price Prediction Neural Network by implementing Early Stopping (to prevent overfitting) and adding an additional hidden layer (to increase model capacity). Three models were trained, evaluated, and compared to identify the best-performing architecture.

Objectives / Purpose: Compare three models — Original, Original + Early Stopping, and Early Stopping + Additional Hidden Layer — using MAE, MSE, RMSE, and R² metrics. Select the best model and update the Anvil web app for improved predictions.

Context Note: This project has two prerequisites: Midterm - Hands-on Long Quiz 1 (Boston House Price Prediction) where the baseline Neural Network model was originally developed and trained, and Midterm - Hands-on Quiz 1 (House Price Prediction) where the model was deployed as a functional web app using Anvil.works. This current project builds upon both by applying model improvement techniques and updating the deployed web app with the best-performing model.

✨ Key Features

  • Three Model Comparison: Systematic evaluation of architectural changes.
  • Early Stopping Implementation: Patience=10, monitor='val_loss', restore_best_weights=True.
  • Additional Hidden Layer: Architecture: 32 → 16 → 1 (compared to original 16 → 8 → 1).
  • Evaluation Metrics: MAE, MSE, RMSE, R² for all three models.
  • Updated Anvil Deployment: Best model deployed to existing web app with 3 test predictions (High, Mid, Low value areas).

💻 Important Code Blocks

1. Early Stopping Callback

Stops training when validation loss stops improving for 10 epochs, restoring best weights to prevent overfitting.


from tensorflow.keras.callbacks import EarlyStopping

early_stop = EarlyStopping(
    monitor='val_loss',
    patience=10,
    restore_best_weights=True
)

history = model.fit(
    X_train, y_train,
    epochs=300,
    validation_split=0.2,
    callbacks=[early_stop]
)

2. Model with Additional Hidden Layer

Increased capacity: 32 neurons in first hidden layer, then 16, then output.


model = Sequential()
model.add(Dense(32, activation='relu', input_shape=(X_train.shape[1],)))
model.add(Dense(16, activation='relu'))  # additional hidden layer
model.add(Dense(1))  # output layer

model.compile(optimizer='adam', loss='mse', metrics=['mae'])

Exposes the best-performing model to Anvil for web app predictions.

📊 Evaluation Metrics & Comparison

Model MAE MSE RMSE R²
Original Model 2.42 15.94 3.99 0.78
Original + Early Stopping 2.62 18.02 4.24 0.75
Early Stopping + Additional Hidden Layer 2.73 17.41 4.17 0.76

Key Observations:

  • Original Model (Best): Lowest MAE (2.42), MSE (15.94), RMSE (3.99), and highest R² (0.78). Had optimal parameter combination.
  • Early Stopping Only (Weakest): Highest errors (MAE 2.62, RMSE 4.24, R² 0.75). Relied solely on early stopping without ideal fit parameters.
  • Early Stopping + Hidden Layer (Second Best): Improved over early stopping-only (R² 0.76) due to increased learning capacity, but slightly worse than original due to added complexity.

Winner: Original Model — deployed to Anvil web app.

🧩 Challenges & Solutions

Challenge 1: Early stopping alone produced worse metrics (R² 0.75) than original model (R² 0.78).

Solution: Recognized that early stopping without matching original parameters (batch_size, verbose) affects learning. Added hidden layer to compensate.

Challenge 2: Additional hidden layer improved over early stopping-only (R² 0.76) but still didn't beat original.

Solution: Concluded original architecture (16→8→1) with full epoch training was optimal for this dataset. Documented as finding.

Challenge 3: Updating Anvil web app with best model without breaking existing functionality.

Solution: Retrained original model with fixed random seed, reconnected Anvil server, tested all three prediction samples.

🧠 Learning Reflection

What I learned: Early stopping prevents overfitting but may stop prematurely if validation loss fluctuates. Adding hidden layers increases model capacity but can introduce unnecessary complexity if the dataset isn't large/complex enough. The original model (16→8→1, 300 epochs) performed best because its architecture matched the dataset's complexity and had optimal training parameters.

Skills gained: Implementing EarlyStopping callbacks in Keras, modifying neural network architectures (adding layers), comparing models systematically using MAE/MSE/RMSE/R², and deploying the best-performing model to Anvil with updated predictions.
